# 2026-03-15 工作日志

## 修复推理脚本 (inference.py) checkpoint 与 dataset 不匹配问题

- **问题**: 用户运行 `python3 inference.py --dataset alibaba --checkpoint checkpoints/best_model.pt` 时报错 `TypeError: forward() got an unexpected keyword argument 'target_item_id'`
- **根因**: `checkpoints/best_model.pt` 是合成数据集训练的 MixFormer 模型（567M 参数），但用户指定了 `--dataset alibaba`。加载时模型被正确识别为 MixFormer（合成数据集），但 dataset_type 被 `--dataset alibaba` 覆盖，导致 Alibaba 格式的数据传入了不兼容的 MixFormer.forward()
- **修复内容**:
  1. 增加 checkpoint 文件存在性检查，不存在时列出可用的 checkpoint 文件
  2. 当 `--dataset` 与 checkpoint 的 `dataset_type` 不匹配时，自动在 `checkpoints/` 和 `checkpoints_alibaba/` 目录下搜索匹配的 checkpoint
  3. 找到匹配的 checkpoint 后自动加载正确的模型
  4. 找不到时回退到 checkpoint 的原始 dataset_type 并给出警告
- **验证**: 两种路径均正常工作（不匹配时自动修正 + 直接指定正确路径）

## 重构：去除合成数据集，默认使用 Alibaba 数据集

  3. `mixformer/data.py` — 去除 `SyntheticRecDataset`、合成数据 `collate_fn`、`create_dataloader`；将 `alibaba_collate_fn` 重命名为 `collate_fn`，`create_alibaba_dataloader` 重命名为 `create_dataloader`
  4. `train.py` — 去除 synthetic/UIMixFormer 分支，默认 Alibaba 训练；`--dataset` 参数改为 `--config default/medium`
  5. `inference.py` — 去除 synthetic/decoupled 分支，简化为纯 Alibaba 推理；兼容旧 checkpoint 中的 `dataset_type` 字段
  6. `mixformer/__init__.py` — 更新导出列表，版本号升至 0.3.0
  7. `tests/test_model.py` — 全部改为 Alibaba 格式数据测试
  8. `README.md` — 去除合成数据集相关文档
- **验证**: 36 个单元测试全部通过，推理脚本正常（checkpoint + 随机权重两种模式均验证通过）
- **兼容性**: `load_model_from_checkpoint` 过滤旧 checkpoint 中已移除的 `dataset_type`/`vocab_sizes` 字段

## 修复训练脚本 MPS (Apple Silicon) 兼容性问题

- **问题**: `python3 train.py --data_dir data/alibaba --epochs 5 --batch_size 4096` 在 Mac 上运行时，`--device auto` 选择 MPS 设备，但 `nn.EmbeddingBag` 不支持 MPS (`aten::_embedding_bag` 未实现)，导致训练崩溃
- **修复内容**:
  1. `mixformer/model.py` — 将 `EmbeddingBagFallback` 中的 `nn.EmbeddingBag` 替换为 `nn.Embedding` + 手动 mean pooling，完全兼容 MPS/CUDA/CPU 所有设备
  2. `train.py` / `inference.py` — 改进 `get_device()` 函数，增加 MPS 检测日志
- **验证**: 36 个单元测试全部通过，MPS 上训练成功（1 epoch, Loss: 0.721, Val AUC: 0.507），推理正常
